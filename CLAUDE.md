# CLAUDE.md

## プロジェクトの目的

犬同伴OK店（わんグルのデータ）をもとに、現在地や地名の周辺で犬と一緒に入れるお店を巡る散歩コースを地図上に提案する静的サイト。サーバーサイド処理なし、GitHub Pagesでの公開を前提とする。

## 技術スタック / 主要ライブラリ

- HTML / CSS / JavaScript（バニラJS、フレームワーク・ビルドツールなし）
- Leaflet.js — 地図描画（`vendor/leaflet/`にバージョン1.9.4を同梱。CDNは使わない。理由は「コース生成方針」の「ルーティングAPIは使わない」と同じ「完全無料・アクセスが増えても止まらない」原則）
- OpenStreetMap — 地図タイル
- Nominatim — 地名検索（ジオコーディングAPI。使うのは検索時の地名→座標変換のみ）
- Geolocation API（ブラウザ標準）— 現在地取得
- Node.js — 開発・テスト時のみ使用（`node --test` でコース生成ロジックを単体テスト）。本番の配信物はNode不要な静的ファイルのみ
- ルーティングAPI（OSRM/GraphHopper等）は**使わない**（詳細は「コース生成方針」参照）

## ディレクトリ構成の方針

```
data/
  shops.json   # 犬同伴OK店データ（2,085件、配置済み・編集不要）
  areas.json   # エリア階層データ（1,163件、配置済み・編集不要）
index.html     # エントリーポイント
css/           # スタイル
js/            # アプリケーションロジック
vendor/
  leaflet/     # Leaflet 1.9.4 を同梱（CDN不使用）
```

- `data/`配下のJSONは既存データであり、原則として書き換えない（スキーマは[README.md](README.md)参照）
- ビルドステップを持たないため、GitHub Pagesでそのまま配信できる構成を維持する
- `js/`内はロジックとUIを分離する（例: `js/route.js`にコース生成ロジック、`js/main.js`に地図描画・イベントハンドリング）。テストは`js/route.test.js`のように対象ファイルと対にする

## コース生成方針（確定設計）

### ルーティングAPIは使わない（重要）

- OSRM / GraphHopper等の経路探索APIは使わない。無料枠・APIキー・利用制限があり、「完全無料・アクセスが増えても止まらない」という必須条件に反するため
- 道路に沿った経路は引かない。店と店は直線で結ぶ
- 外部アクセスは「地名→座標変換（Nominatim）」の検索時1回のみ。それ以外は同梱JSON（`data/shops.json` / `data/areas.json`）を読むだけ

### 入力

- 「現在地から探す」ボタン（Geolocation API）
- または「地名・駅名を入力」欄 → Nominatimで座標化
  - Nominatimには User-Agent を名乗り、1秒1回以上叩かないマナーを守る

### コース生成ロジック

- 入力地点から半径2kmで犬同伴OK店を探す。2店未満なら5km→10km…と自動で範囲を広げる
- 入力地点に一番近い店を起点に、最近傍を順にたどる数珠つなぎで2〜4店を選ぶ
- 同座標の店（地図埋め込みのコピペにより`data/shops.json`に37組=74店存在）はコースに1店だけ採用し、残りは「近くにこんな店も」として別枠で紹介する
- 距離はハバーサイン公式による直線距離で計算。徒歩時間は直線距離 ÷ 分速80mの概算
- 徒歩時間・距離は「目安」であることをUI上で明示する（実際は道なりで直線距離より長くなるため）

### 表示

- 地図（Leaflet + OpenStreetMapタイル）に①②③…の番号つきピン + 店を結ぶ直線
- 各店カード: 店名・ジャンル・最寄駅（`access`）・直線距離・約○分（目安）・「詳しくは→わんグル」（`url`へのリンク）
- コース全体の合計距離・所要時間の目安

### スコープ（v1で入れないもの）

- エリア絞り込み・ジャンル絞り込みはv1では入れない（`areas.json`は同梱のみ。後日の拡張用）

### 設計の分離（テスト方針）

- コース生成ロジック（距離計算・数珠つなぎ・コピペ地図の重複除外・範囲自動拡張）は純粋関数として切り出し、UI（DOM操作・地図描画）から分離する
- シグネチャの目安: 「入力座標 + 店リスト」→「順番つきのコース（数店）」を返す純粋関数
- テストは`node --test`（Node.js組み込みテストランナー）で実行する

## デザイン方針（世界観）

姉妹プロダクト「散歩危険度チェッカー」「わんワード」と統一した、穏やかな世界観に合わせる。

- アクセントカラー: `#3D8F63`
- カード: 角丸16px
- ボタン: 角丸999px（ピル型）
- 背景: クリーム系 `#FDF8F2`
- 犬のイラストをあしらう

## コーディング規約

- ES modules（`<script type="module">`）を基本とし、外部フレームワークは導入しない
- インデントは2スペース、文字コードはUTF-8、改行はLF（`.editorconfig`参照）
- DOM操作・地図初期化・データ取得（fetch）は関数単位で分離し、グローバル変数を増やさない

## よく使うコマンド

```bash
# ローカルプレビュー（fetchでJSONを読むため簡易サーバー経由で開く）
python -m http.server 8000
# または
npx serve .

# コース生成ロジックの単体テスト（js/route.js等の純粋関数が対象）
node --test
```

Lint/型チェックツールは未導入。コース生成ロジック（距離計算・数珠つなぎ等の純粋関数）は`node --test`でテストし、UI（地図描画・Geolocation・Nominatim検索）はブラウザでの目視確認を基本とする。

## 注意事項

- `.env`は読まない・コミットしない（本プロジェクトはNominatim/OpenStreetMap/Geolocationのみを使い、APIキーは不要）
- Nominatimの利用ポリシー上、リクエストは1秒間に1回程度に抑え、過度な連続リクエストを避ける
- 地図・データの出典表示（OpenStreetMapの著作権表示、わんグル由来である旨）を画面上に残す
- `data/shops.json`の`note`にある通り、店舗の`lat`/`lng`は店舗ページの地図埋め込み中心座標であり、実際のピン位置とずれる場合がある前提でUIを設計する

※ コードが揃ったら `/init` で本ファイルを更新・拡充できます。
